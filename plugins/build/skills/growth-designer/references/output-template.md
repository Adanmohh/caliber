# Growth Designer — Output Template

## Source Attribution
- Engagement Audit: Based on Nir Eyal's Hooked Model framework
- Habit Loop Design: Hooked Model + BJ Fogg Behavior Model
- Onboarding Flow: Wes Bush's Bowling Alley Framework
- PLG Blueprint: Wes Bush's Product-Led Growth + MOAT Framework
- Opportunity Solution Tree: Teresa Torres' Continuous Discovery Habits
- Retention Risk Assessment: Composite of all three frameworks

---

## How To Use This Template

This template provides structured output formats for every major deliverable
The Growth Designer produces. Each section can be used independently or
combined into a comprehensive growth design document.

When generating output, the AI agent system should:
1. Select the relevant template sections based on the user's request
2. Fill in every field with specific, actionable content (no placeholders)
3. Include scoring where applicable
4. Attach quality checklist at the end
5. Reference the specific framework being applied

## Stage-Gated Guidance

Adapt outputs to the venture's current stage. For pre-launch: use hypothesis-driven language, note where user research is needed, and avoid assuming conversion metrics or cohort data exist.

---

## Quality Checklist

Apply this checklist to every Growth Designer output before delivery.

```
QUALITY CHECKLIST
=================

COMPLETENESS:
[ ] All relevant template sections filled (no blanks or placeholders)
[ ] Specific numbers, not ranges (current metrics, targets, scores)
[ ] Real examples referenced (not hypothetical)
[ ] Framework explicitly cited (Torres / Eyal / Bush)

ACTIONABILITY:
[ ] Every recommendation has a specific next action
[ ] Priority order defined (what to do first)
[ ] Success metrics defined for each recommendation
[ ] Effort estimates included (time, resources)

EVIDENCE:
[ ] Customer interview data referenced where available
[ ] Behavioral data cited where available
[ ] Benchmarks sourced from industry data
[ ] Assumptions explicitly called out

ETHICAL:
[ ] Manipulation Matrix assessment completed
[ ] Facilitator quadrant confirmed (or issue flagged)
[ ] User autonomy preserved in all designs
[ ] Dark patterns explicitly avoided

COHERENCE:
[ ] Hook cycle phases connect logically
[ ] Investment loads next trigger (verified)
[ ] Onboarding flow leads to aha moment (verified)
[ ] PLG metrics align with business model
[ ] OST connects outcome -> opportunity -> solution -> test

HANDOFF READINESS:
[ ] All handoff documents completed for relevant recipients
[ ] Instrumentation requirements specified for engineering
[ ] Design requirements specified for product designer
[ ] No ambiguous requirements (specific enough to implement)
```
