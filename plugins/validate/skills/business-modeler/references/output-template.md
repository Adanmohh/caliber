# The Business Modeler: Output Template

> **Stage-Gated Guidance:** Adapt outputs to the venture's current stage. For pre-launch/idea-stage ventures: use simulated data clearly labeled [SIMULATED], focus on assumptions over metrics, and note where real data is needed before acting.

This template defines the structured output format for all Business Modeler deliverables.
Every business model analysis must include ALL sections below, filled with specific
content — no placeholder text or "TBD" entries in the final output.

---

## OUTPUT STRUCTURE OVERVIEW

Every Business Modeler output includes these sections in order:

```
1. Executive Summary
2. Business Model Canvas (all 9 blocks)
3. Strategy Canvas (value curve analysis)
4. Four Actions Grid (ERRC)
5. Key Assumptions Risk Matrix
6. Financial Sketch
7. Business Model Pattern Classification
8. Non-Customer Analysis
9. Quality Checklist (self-assessment)
10. Handoff to Experiment Designer
```

## FORMATTING RULES

1. Use markdown tables for all structured data (no prose where tables work)
2. Use specific numbers, not ranges, wherever possible (ranges are acceptable
   only when genuine uncertainty exists)
3. Score everything — no unscored factors, no unranked risks
4. Cite sources for benchmarks and industry data (URL or publication name)
5. Use active voice: "Customers pay $X/month" not "Revenue is generated through..."
6. Every assumption must be FALSIFIABLE — if it can't be tested, rewrite it
7. Taglines and summaries must fit in one sentence
8. All financial estimates must state their basis: "[X] based on [source/method]"
9. Version the output: include a version number and date
10. ASCII charts preferred over descriptions of charts — show, don't tell
