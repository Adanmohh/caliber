# Business Model Canvas: Complete Reference

Source: Alexander Osterwalder & Yves Pigneur, "Business Model Generation" (2010).
Additional sources: Strategyzer methodology, Ash Maurya "Running Lean" (2012),
Steve Blank "The Startup Owner's Manual" (2012).

---

## 1. WHAT IS THE BUSINESS MODEL CANVAS?

The Business Model Canvas (BMC) is a one-page strategic tool that maps how a
business creates, delivers, and captures value. It consists of 9 interdependent
building blocks arranged in a specific visual layout.

**Critical principle:** Every element on the canvas is a HYPOTHESIS until validated.
The canvas is not a plan — it is a set of testable assumptions organized for
systematic de-risking.

### Canvas Layout (ASCII)

```
+------------------+------------------+------------------+------------------+------------------+
|                  |                  |                  |                  |                  |
|  KEY             |  KEY             |  VALUE           |  CUSTOMER        |  CUSTOMER        |
|  PARTNERSHIPS    |  ACTIVITIES      |  PROPOSITIONS    |  RELATIONSHIPS   |  SEGMENTS        |
|  [8]             |  [7]             |  [1]             |  [4]             |  [2]             |
|                  |                  |                  |                  |                  |
|                  +------------------+                  +------------------+                  |
|                  |                  |                  |                  |                  |
|                  |  KEY             |                  |  CHANNELS        |                  |
|                  |  RESOURCES       |                  |  [3]             |                  |
|                  |  [6]             |                  |                  |                  |
|                  |                  |                  |                  |                  |
+------------------+------------------+------------------+------------------+------------------+
|                                     |                                     |
|  COST STRUCTURE [9]                 |  REVENUE STREAMS [5]                |
|                                     |                                     |
+-------------------------------------+-------------------------------------+
```

The numbers in brackets indicate the RECOMMENDED FILL ORDER (see Section 3).

## 2. THE 9 BUILDING BLOCKS — DETAILED

### BLOCK 1: VALUE PROPOSITIONS

**Definition:** The bundle of products and services that create value for a specific
customer segment. This is the reason customers choose one company over another.

**Value Creation Levers (Osterwalder's 11 types):**

| # | Lever             | Description                                  | Example                        |
|---|-------------------|----------------------------------------------|--------------------------------|
| 1 | Newness           | Satisfying entirely new needs                | iPhone (2007)                  |
| 2 | Performance       | Improving product/service performance        | Intel processors               |
| 3 | Customization     | Tailoring to individual needs                | Nike By You                    |
| 4 | Getting the job done | Helping customers complete a task          | Rolls-Royce "Power by Hour"    |
| 5 | Design            | Superior aesthetic or functional design      | Bang & Olufsen                 |
| 6 | Brand/Status      | Signaling identity through brand             | Rolex, Tesla                   |
| 7 | Price             | Offering comparable value at lower price     | Xiaomi phones                  |
| 8 | Cost reduction    | Helping customers reduce their costs         | Salesforce vs on-premise CRM   |
| 9 | Risk reduction    | Reducing risk associated with purchase       | 30-day money-back guarantees   |
|10 | Accessibility     | Making available what was previously unavail | Robinhood (stock trading)      |
|11 | Convenience       | Making things easier or more convenient      | Uber vs traditional taxis      |

**AI Agent Execution:** When filling this block, research the specific value levers
competitors are using. Score each lever 1-5 for current industry emphasis, then
identify gaps where no competitor scores above 2. These gaps are opportunity zones.

**Diagnostic Questions:**
1. What job is the customer hiring this product to do?
2. Which customer pain does this eliminate or reduce?
3. Which customer gain does this create or amplify?
4. Is the value proposition a "painkiller" (must-have) or a "vitamin" (nice-to-have)?

**Scoring System for Value Proposition Strength:**
- 5 = Must-have, customers would pay 3x current price
- 4 = Strong differentiator, clear switching motivation
- 3 = Competitive parity, matches alternatives
- 2 = Weak, customers have better options
- 1 = No real value, hypothetical benefit only

### BLOCK 2: CUSTOMER SEGMENTS

**Definition:** The different groups of people or organizations a business aims to
reach and serve.

**Five Segment Types (Osterwalder):**

1. **Mass Market** — No distinction between segments (consumer electronics)
2. **Niche Market** — Specific, specialized segments (luxury watches)
3. **Segmented** — Slightly different needs within broad market (bank retail vs private)
4. **Diversified** — Serves unrelated segments (Amazon: retail + AWS)
5. **Multi-sided Platform** — Serves 2+ interdependent segments (Google: users + advertisers)

**Segmentation Variables:**

| Category       | Variables                                           |
|----------------|-----------------------------------------------------|
| Demographics   | Age, income, education, occupation, family size     |
| Firmographics  | Industry, company size, revenue, location           |
| Behavioral     | Usage frequency, loyalty status, purchase occasion  |
| Psychographic  | Values, lifestyle, attitudes, personality           |
| Needs-based    | Job-to-be-done, pain severity, gain importance      |

**AI Agent Execution:** Research actual market size data for each segment candidate.
Use web research to find industry reports, census data, and competitor targeting
patterns. Rank segments using the Segment Attractiveness Matrix below.

**Segment Attractiveness Matrix (score each 1-5):**

| Factor                  | Weight | Score | Weighted Score |
|-------------------------|--------|-------|----------------|
| Market size             | 0.20   |       |                |
| Growth rate             | 0.15   |       |                |
| Willingness to pay      | 0.20   |       |                |
| Accessibility           | 0.15   |       |                |
| Competition intensity   | 0.15   |       |                |
| Strategic fit           | 0.15   |       |                |
| **TOTAL**               | 1.00   |       |                |

Select the segment with the highest weighted score as the beachhead.

### BLOCK 3: CHANNELS

**Definition:** How a company communicates with and reaches its customer segments
to deliver the value proposition.

**Channel Phases (Osterwalder's 5 phases):**

1. **Awareness** — How do customers learn about the product?
2. **Evaluation** — How do customers evaluate the value proposition?
3. **Purchase** — How do customers buy?
4. **Delivery** — How is the value proposition delivered?
5. **After-Sales** — How is post-purchase support provided?

**Channel Types:**

| Type          | Own/Partner | Examples                              |
|---------------|-------------|---------------------------------------|
| Direct Sales  | Own         | Sales force, website, app             |
| Indirect      | Partner     | Retail stores, wholesalers            |
| Digital       | Own         | E-commerce, SaaS platform            |
| Affiliate     | Partner     | Referral programs, affiliate networks |
| Hybrid        | Both        | Apple (own stores + Best Buy)         |

**Channel Efficiency Scoring:**

Score each channel on: Reach (1-5), Cost-efficiency (1-5), Control (1-5),
Customer experience (1-5). Calculate weighted average. Channels scoring below
3.0 overall should be flagged for redesign.

---

### BLOCK 4: CUSTOMER RELATIONSHIPS

**Definition:** The types of relationships a company establishes with specific
customer segments.

**Six Relationship Types:**

1. **Personal Assistance** — Human interaction (bank tellers, account managers)
2. **Dedicated Personal Assistance** — Assigned relationship manager (wealth management)
3. **Self-Service** — No direct relationship, all tools provided (IKEA)
4. **Automated Services** — Automated + personalized (Netflix recommendations)
5. **Communities** — User communities for peer support (Salesforce Trailblazers)
6. **Co-creation** — Customer participates in value creation (YouTube, Lego Ideas)

**Relationship-Segment Alignment Check:**
- High-value segments -> Dedicated personal assistance
- Mass market -> Self-service or automated
- Tech-savvy segments -> Communities + self-service
- Creative segments -> Co-creation

**Key Metrics per Relationship Type:**
- Acquisition cost (CAC) by relationship type
- Retention rate by relationship type
- Lifetime value (LTV) by relationship type
- LTV:CAC ratio target: minimum 3:1

### BLOCK 5: REVENUE STREAMS

**Definition:** The cash a company generates from each customer segment.

**Seven Revenue Models:**

| # | Model                | Description                                    | Example                    |
|---|----------------------|------------------------------------------------|----------------------------|
| 1 | Asset sale           | Selling ownership of a physical product        | Apple selling iPhones      |
| 2 | Usage fee            | Charging for use of a service                  | AWS compute hours          |
| 3 | Subscription fee     | Recurring access to a service                  | Netflix, Spotify           |
| 4 | Lending/Leasing      | Temporary access to an asset                   | Zipcar, equipment leasing  |
| 5 | Licensing            | Permission to use intellectual property        | Microsoft Office licenses  |
| 6 | Brokerage fee        | Intermediation fee for a transaction           | Real estate agents, Stripe |
| 7 | Advertising          | Charging for attention/impressions             | Google, Facebook           |

**Pricing Mechanisms:**

| Mechanism           | Description                         | Best For                    |
|---------------------|-------------------------------------|-----------------------------|
| Fixed list pricing  | Set prices based on features/volume | Standardized products       |
| Dynamic pricing     | Prices change based on conditions   | Airlines, hotels, ride-share|
| Auction             | Price determined by bidding         | Art, ad placements          |
| Market-dependent    | Price set by supply/demand          | Commodities                 |
| Volume-dependent    | Discounts for quantity              | Enterprise software, SaaS   |
| Negotiation         | Price negotiated per deal           | B2B enterprise sales        |

**Revenue Stream Health Checklist:**
- [ ] Multiple revenue streams (diversification)
- [ ] At least one recurring revenue stream
- [ ] Revenue stream matches customer willingness to pay
- [ ] Pricing mechanism fits buyer behavior
- [ ] Gross margin exceeds industry benchmark

### BLOCK 6: KEY RESOURCES

**Definition:** The most important assets required to make the business model work.

**Four Resource Categories:**

1. **Physical** — Buildings, vehicles, machines, distribution networks
2. **Intellectual** — Brands, patents, proprietary knowledge, databases, partnerships
3. **Human** — Specialized skills, domain expertise, creative talent
4. **Financial** — Cash reserves, credit lines, stock option pools, revenue guarantees

**Resource Criticality Assessment:**

For each resource, score:
- Substitutability (1=easily replaced, 5=irreplaceable)
- Contribution to VP (1=marginal, 5=essential)
- Competitive advantage (1=commodity, 5=unique moat)

Resources scoring 12+ out of 15 are CRITICAL. Losing them would break the model.

---

### BLOCK 7: KEY ACTIVITIES

**Definition:** The most important things a company must do to make its business
model work.

**Three Activity Categories (Osterwalder):**

1. **Production** — Designing, making, delivering (manufacturing companies)
2. **Problem Solving** — Coming up with new solutions (consultancies, hospitals)
3. **Platform/Network** — Maintaining and developing platforms (eBay, Visa)

**Activity Prioritization Matrix:**

| Activity            | Value Impact (1-5) | Cost (1-5) | Outsourceable? | Priority |
|---------------------|--------------------|------------|----------------|----------|
| [Activity name]     |                    |            | Y/N            | H/M/L    |

Activities with high value impact AND low outsourceability = keep in-house.
Activities with low value impact AND high outsourceability = outsource or automate.

---

### BLOCK 8: KEY PARTNERSHIPS

**Definition:** The network of suppliers and partners that make the business model work.

**Four Partnership Types:**

1. **Strategic alliances** between non-competitors (Starbucks + Spotify)
2. **Coopetition** — partnerships between competitors (Samsung making Apple displays)
3. **Joint ventures** to develop new businesses (Sony Ericsson)
4. **Buyer-supplier relationships** for reliable supply (Toyota supplier keiretsu)

**Three Motivations for Partnerships:**

1. **Optimization and economies of scale** — Reduce cost via shared infrastructure
2. **Risk and uncertainty reduction** — Share R&D costs, hedge against disruption
3. **Acquisition of resources/activities** — Access capabilities you don't have

**Partnership Risk Assessment:**
- Dependency level (1-5): How dependent are you on this partner?
- Switching cost (1-5): How hard is it to replace them?
- Alignment (1-5): How aligned are your strategic interests?

Partners with dependency 4+ and switching cost 4+ are HIGH RISK. Develop backup plans.

---

### BLOCK 9: COST STRUCTURE

**Definition:** All costs incurred to operate the business model.

**Two Cost Structure Types:**

1. **Cost-driven** — Minimize costs wherever possible (Ryanair, Walmart, IKEA)
2. **Value-driven** — Focus on premium value creation (Four Seasons, Rolex)

**Cost Characteristics:**

| Characteristic       | Description                                   |
|----------------------|-----------------------------------------------|
| Fixed costs          | Don't change with volume (rent, salaries)     |
| Variable costs       | Scale with production volume (materials, commissions) |
| Economies of scale   | Cost per unit decreases as volume increases   |
| Economies of scope   | Cost per unit decreases as product range expands |

**Unit Economics Formula:**
```
Contribution Margin = Revenue per Unit - Variable Cost per Unit
Break-even Volume = Fixed Costs / Contribution Margin
```

**Cost Structure Health Check:**
- Fixed-to-variable ratio (higher variable = more flexibility)
- Gross margin vs industry benchmark
- Customer acquisition cost (CAC) as % of first-year revenue
- Time to break even on CAC

## 3. RECOMMENDED FILL ORDER

The canvas should NOT be filled left-to-right or randomly. Use this priority order:

```
PRIORITY 1 (Start here — defines the business):
  [1] Value Propositions — What value do you deliver?
  [2] Customer Segments — For whom do you create value?

PRIORITY 2 (Connects value to customers):
  [3] Channels — How do you reach customers?
  [4] Customer Relationships — What relationship do they expect?
  [5] Revenue Streams — How do you capture value?

PRIORITY 3 (What it takes to deliver):
  [6] Key Resources — What do you need?
  [7] Key Activities — What must you do?
  [8] Key Partnerships — Who helps you?

PRIORITY 4 (Financial reality):
  [9] Cost Structure — What does it cost?
```

**Why this order matters:** Starting with Value Proposition + Customer Segment
forces outside-in thinking. Starting with Resources or Activities (common mistake)
leads to inside-out thinking that ignores customer reality.

## 4. DEPENDENCIES BETWEEN BLOCKS

Understanding block dependencies prevents internal contradictions:

```
Customer Segments ---determines---> Value Propositions
Value Propositions ---determines---> Channels (how to deliver the VP)
Customer Segments ---determines---> Customer Relationships (what they expect)
Value Propositions + Customer Segments ---determines---> Revenue Streams
Revenue Streams ---constrains---> Cost Structure (must be profitable)
Value Propositions ---requires---> Key Resources + Key Activities
Key Activities ---may require---> Key Partnerships
Key Resources + Key Activities + Key Partnerships ---determines---> Cost Structure
```

**Consistency Check Questions:**
1. Does every customer segment have a matching value proposition?
2. Does every channel serve at least one customer segment?
3. Do revenue streams match customer willingness to pay?
4. Do key resources enable the stated key activities?
5. Does cost structure allow for profitable revenue capture?

## 5. BUSINESS MODEL PATTERNS

### Pattern 1: UNBUNDLING (Source: John Hagel III)

Three fundamentally different business types that should not be mixed:
- **Product Innovation** — Speed to market, talent-driven (Apple product design)
- **Customer Relationship** — Scope economies, trust-driven (retail banking)
- **Infrastructure Management** — Scale economies, cost-driven (telecom networks)

**Diagnostic:** If your canvas tries to do all three, it has internal conflicts.
Unbundle into separate models or choose your primary type.

### Pattern 2: LONG TAIL (Source: Chris Anderson, 2006)

Sell less of more — aggregate many niche products rather than few bestsellers.

**Canvas signature:**
- Value Proposition: Large catalog of niche items
- Key Resources: Platform infrastructure, recommendation algorithms
- Revenue Streams: Many small transactions
- Cost Structure: Low inventory cost per item

**Examples:** Amazon Marketplace, Netflix content library, iTunes/App Store,
Spotify's catalog of 100M+ tracks

**Viability test:** Do you have (a) low per-item cost, (b) strong discovery/search
mechanisms, and (c) aggregate demand that exceeds hit-driven alternatives?

### Pattern 3: MULTI-SIDED PLATFORMS (Source: Jean-Charles Rochet & Jean Tirole)

Bring together two or more distinct customer groups who need each other.

**Canvas signature:**
- Customer Segments: 2+ interdependent groups
- Value Proposition: Different VP per side (matchmaking + tools)
- Revenue Streams: One side subsidized, one side monetized
- Key Activities: Platform development, network effects management

**Examples:**
- Google (users get free search, advertisers pay for ads)
- Uber (riders get rides, drivers get income, Uber takes commission)
- Visa (cardholders get convenience, merchants get customer access)

**Critical success factor:** Solve the chicken-and-egg problem. Which side do you
subsidize to attract the other side? The side with higher price sensitivity and
higher cross-side network effects should be subsidized.

### Pattern 4: FREEMIUM (Source: Fred Wilson, 2006)

Free basic service with premium paid upgrades.

**Canvas signature:**
- Customer Segments: Free users (large base) + Premium users (small, paying)
- Value Proposition: Basic = "good enough," Premium = "power features"
- Revenue Streams: Premium subscriptions (typically 2-5% conversion rate)
- Cost Structure: Marginal cost of free users must be near zero

**Conversion Rate Benchmarks:**
- Consumer freemium: 2-5% conversion (Spotify, Dropbox)
- B2B freemium: 5-15% conversion (Slack, Zoom)
- Gaming freemium: 1-3% conversion (mobile games)

**Examples:** Spotify (free with ads vs Premium), Dropbox (2GB free vs paid),
LinkedIn (basic vs Premium), Slack (free vs paid teams)

**Viability test:** Can you serve free users at near-zero marginal cost while
delivering enough premium value that 2-5% convert?

### Pattern 5: RAZOR AND BLADE (Source: King Gillette, 1901)

Sell the base product cheaply, profit from consumables.

**Canvas signature:**
- Value Proposition: Low barrier to entry for base product
- Revenue Streams: Recurring consumable purchases at high margin
- Key Resources: Lock-in mechanism (proprietary consumables)
- Cost Structure: Loss or break-even on base product

**Examples:** Gillette (cheap razor, expensive blades), Nespresso (machine +
capsules), HP (printers + ink cartridges), Xbox/PlayStation (console + games)

**Reverse razor-and-blade:** Apple charges premium for hardware (iPhone) but
offers apps/services at accessible prices or free (iMessage, FaceTime).

### Pattern 6: OPEN BUSINESS MODEL (Source: Henry Chesbrough)

Use external ideas to create value (outside-in) or allow internal ideas to be
used externally (inside-out).

**Examples:**
- Outside-in: Procter & Gamble's "Connect + Develop" — 50% of innovations from
  external sources
- Inside-out: Tesla open-sourcing patents to grow the EV market

## 6. ASSUMPTION RISK ASSESSMENT

Every element on the canvas is a hypothesis. Rank each assumption using the
Assumption Risk Matrix:

**Risk Score = Importance (1-5) x Uncertainty (1-5)**

| Risk Score | Category        | Action Required                            |
|------------|-----------------|--------------------------------------------|
| 20-25      | Critical Risk   | Test IMMEDIATELY before any other work     |
| 15-19      | High Risk       | Test within first sprint/cycle             |
| 10-14      | Medium Risk     | Test within next sprint                    |
| 5-9        | Low Risk        | Monitor, test when resources allow         |
| 1-4        | Negligible Risk | Accept the risk; do not allocate resources |

**Importance Scale:**
- 5 = If wrong, entire business model fails
- 4 = If wrong, major pivot required
- 3 = If wrong, significant redesign of 1-2 blocks
- 2 = If wrong, minor adjustment needed
- 1 = If wrong, negligible effect

**Uncertainty Scale:**
- 5 = Pure guess, no data at all
- 4 = Anecdotal evidence only
- 3 = Some data but not from target segment
- 2 = Reasonable data from comparable situations
- 1 = Validated by direct evidence

**Testing Methods by Block:**

| Block               | Testing Method                                    |
|---------------------|---------------------------------------------------|
| Value Proposition   | Customer interviews, landing page tests, prototypes|
| Customer Segments   | Surveys, TAM/SAM/SOM analysis, persona validation |
| Channels            | A/B testing, pilot partnerships, conversion funnels|
| Customer Relations  | Support ticket analysis, churn studies, NPS surveys|
| Revenue Streams     | Willingness-to-pay studies, price sensitivity tests|
| Key Resources       | Capability audits, build-vs-buy analysis          |
| Key Activities      | Process mapping, time-motion studies               |
| Key Partnerships    | LOI negotiations, pilot programs                   |
| Cost Structure      | Bottom-up cost modeling, comparable company analysis|

## 8. AI AGENT EXECUTION PROTOCOL

When designing a Business Model Canvas as an AI agent:

**Step 1: Context Gathering**
- Research the industry, competitors, and market size via web search
- Identify 3-5 competitor business models for comparison
- Gather pricing data, margin benchmarks, and growth rates

**Step 2: Pattern Matching**
- Compare the business opportunity against all 6 patterns above
- Identify which pattern(s) best fit the context
- Flag any pattern mismatches (e.g., trying freemium with high marginal costs)

**Step 3: Canvas Construction**
- Fill blocks in priority order (Section 3)
- Check dependencies (Section 4) for internal consistency
- Score each assumption using the Risk Matrix (Section 6)

**Step 4: Stress Testing**
- Apply the 5 consistency check questions (Section 4)
- Verify unit economics: LTV > 3x CAC, gross margin > industry benchmark
- Identify the top 3 riskiest assumptions
- Generate 2-3 alternative canvas configurations

**Step 5: Output Generation**
- Produce the completed canvas using the output template
- Attach the assumption risk matrix
- Write clear handoff notes for experiment design

## 9. COMMON FAILURE MODES

| # | Failure Mode                          | Fix                                            |
|---|---------------------------------------|-------------------------------------------------|
| 1 | VP doesn't match segment needs        | Re-interview, run Jobs-to-be-Done analysis      |
| 2 | Revenue model mismatches willingness   | Run Van Westendorp pricing study                |
| 3 | Channels don't reach target segment   | Map customer journey, test alternative channels |
| 4 | Cost structure makes unit economics negative | Redesign, find lower-cost alternatives    |
| 5 | Key resource is a single point of failure | Build redundancy or find substitutes        |
| 6 | Partnership dependency too high       | Develop in-house capability or diversify partners|
| 7 | Canvas is internally contradictory    | Re-run dependency check (Section 4)             |
| 8 | All assumptions are "low risk"        | You're being overconfident — re-score honestly  |
| 9 | Canvas looks like competitor's canvas | You have no differentiation — apply Blue Ocean  |
|10 | Too many customer segments            | Focus on beachhead segment first                |

## 10. CANVAS ITERATION TRIGGERS

The canvas should be updated when:
- Customer interview data contradicts an assumption
- A hypothesis test fails (experiment invalidates a block)
- Market conditions change (new competitor, regulation, technology)
- Unit economics don't work at current scale projections
- A partnership falls through or a new one emerges
- The team pivots on customer segment or value proposition

**Iteration protocol:**
1. Identify which block(s) need updating
2. Check downstream dependencies
3. Re-score affected assumptions
4. Generate updated canvas version (keep version history)
5. Communicate changes to other agents in the system
