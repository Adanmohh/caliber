# $100M Money Models: Leads, Closing, Proof, Delivery

> Source: Alex Hormozi, $100M Money Models (2025). Developed from working with 1,026+ businesses.

---

## The Four Money Model Systems

| System | Purpose | Core Question |
|--------|---------|---------------|
| **Leads** | Get customers in the door | "How do I get more people to want my stuff?" |
| **Closing** | Convert prospects to buyers | "How do I get them to say yes?" |
| **Proof** | Shorten sales cycles, build trust | "How do I make them believe before buying?" |
| **Delivery** | Maximize value per customer | "How do I make more money from each customer?" |

A Money Model = the deliberate sequence of offers: attract with front-end offers, monetize with upsells/downsells, maximize with continuity.

## 1. Leads System

### The Goated Ads Playbook

**The 70/20/10 Framework (creative budget allocation):**

| Allocation | Purpose | Description |
|-----------|---------|-------------|
| 70% | Scale winners | Proven hooks and angles that already convert |
| 20% | Iterate on winners | Variations of winning creative (new hook, same angle) |
| 10% | Test wild cards | Completely new concepts, formats, audiences |

**Ad Assembly by Awareness Level:**

| Awareness | Lead Knows | Ad Strategy |
|-----------|-----------|-------------|
| Unaware | Nothing about the problem | Lead with pain/story; educate on the problem |
| Problem Aware | They have a problem | Agitate the problem; introduce your mechanism |
| Solution Aware | Solutions exist | Differentiate your approach; proof-heavy |
| Product Aware | Your product exists | Overcome objections; offer incentives |
| Most Aware | Your product, ready to buy | Direct CTA; deal/urgency |

### The Hooks Playbook

Distilled from analyzing 35,000+ pieces of content. Best hooks:
- Call out the audience directly ("If you're a [role] struggling with [pain]...")
- Lead with a counterintuitive result ("We stopped [common practice] and revenue went up 34%")
- Use specific numbers ("How I went from $X to $Y in Z days")
- Open with the objection ("This won't work if you [common excuse]... except it did")

## 2. Closing System

### The Seven Universal Closes

| Close | When to Use | Core Script Pattern |
|-------|------------|-------------------|
| **The Ownership Close** | Prospect imagines life with product | "Imagine it's 90 days from now and [dream outcome]..." |
| **The Scarcity Close** | Limited availability (real) | "We only take X clients because [real reason]..." |
| **The Urgency Close** | Time-limited opportunity | "This [pricing/bonus] is available until [date] because..." |
| **The Guarantee Close** | Prospect fears risk | "If [specific outcome] doesn't happen, [guarantee]..." |
| **The Either/Or Close** | Prospect stuck between options | "Would you prefer [Option A] or [Option B]?" |
| **The If-I-Can Close** | Specific objection surfaced | "If I can show you [address objection], would you move forward?" |
| **The Reframe Close** | Price or timing objection | "The cost of NOT doing this is [calculated cost of inaction]..." |

### The Reframe Framework for Objections

```
Objection -> Acknowledge -> Reframe -> Close
"Too expensive" -> "I hear you" -> "What's it costing you NOT to solve this?" -> "If I can..."
```

## 3. Proof System

### The Belief Continuum

Prospects move along a spectrum from skepticism to certainty:

```
SKEPTICAL --> CURIOUS --> INTERESTED --> CONVINCED --> CERTAIN
    |            |            |              |            |
  Needs:     Needs:      Needs:         Needs:       Needs:
  Social     Stats &     Case studies   Guarantees   Just the
  proof at   data        with specifics & direct     offer
  scale      points                     testimonials
```

**Proof types ranked by persuasion strength:**

| Proof Type | Strength | Best For |
|-----------|----------|----------|
| Demo/live results | Highest | Product-aware prospects |
| Named case study with metrics | Very High | Solution-aware prospects |
| Video testimonial | High | All awareness levels |
| Written testimonial with specifics | Medium-High | Landing pages, emails |
| Aggregate data ("2,847 customers") | Medium | Social proof at scale |
| Logos/badges/certifications | Medium | Trust building |
| Screenshots/before-after | Medium | Visual proof |
| Implied proof (design quality, speed) | Lower | Ambient trust signals |

**13-Point Conversion Diagnostic (why marketing assets don't convert):**

Check each: (1) Wrong audience, (2) Weak hook, (3) Unclear offer, (4) No proof, (5) Wrong proof type, (6) Missing guarantee, (7) No urgency, (8) Too many CTAs, (9) Wrong awareness level, (10) Price not anchored, (11) Objections unaddressed, (12) No social proof, (13) Weak follow-up.

## 4. Delivery System

### The 8 Transaction Types

| Type | Description | Revenue Pattern |
|------|------------|----------------|
| **One-time sale** | Single purchase, no recurrence | Spike, then zero |
| **Subscription** | Recurring payment for ongoing access | Predictable monthly |
| **Consumable** | Product used up, repurchased | Repeat purchases |
| **Usage-based** | Pay per use/consumption | Variable, scales with usage |
| **License** | Pay for rights to use | Annual or periodic |
| **Financing** | Payments over time for one product | Spread revenue |
| **Service retainer** | Ongoing service delivery | Predictable monthly |
| **Hybrid** | Combination of above types | Layered revenue |

### Customer Value Maximization Map

```
ACQUISITION --> ACTIVATION --> MONETIZATION --> RETENTION --> EXPANSION
     |              |              |               |              |
  Get them in    First value    Core revenue    Keep them     Sell more
  (Leads System) delivered      (initial sale)  (Delivery)    (Ascension)
                                     |
                              +------+------+
                              |      |      |
                           Upsell  Cross   Continuity
                                   sell
```

### Subscription/Continuity Design (Highest-Value Money Model)

| Design Element | Best Practice |
|---------------|---------------|
| Front-load value | Deliver biggest win in first 30 days |
| Create switching costs | Data lock-in, community bonds, workflow integration |
| Build habits | Daily/weekly usage triggers |
| Milestone rewards | Recognize progress to increase commitment |
| Annual lock-in | Offer 2 months free for annual commitment |
| Expansion revenue | Usage tiers that grow with customer success |

## Decision Tree: Which System Needs Work?

```
Getting enough leads?
  NO --> LEADS SYSTEM (hooks, ads, content)
  YES |
      v
Converting leads to buyers?
  NO --> Is it a trust/belief problem?
         YES --> PROOF SYSTEM (case studies, testimonials, demos)
         NO  --> CLOSING SYSTEM (objection handling, urgency, guarantees)
  YES |
      v
Customers staying and paying enough?
  NO --> DELIVERY SYSTEM (retention, upsells, continuity)
  YES --> Optimize all four. Scale what works. Test new channels.
```
